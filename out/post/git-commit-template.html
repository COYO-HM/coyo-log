git commit template

commit 시에 등록한 commit template을 보여주어 commit message를 작성할 때 일정한 스타일을 유지할 수 있도록 도와줍니다.

git commit template 등록하기

template 파일 생성하고 작성하기

  1. .git 폴더가 있는 위치에 .gitmessage.txt 파일을 생성
  1. 생성한 .gitmessage.txt 에 내용을 작성
  • # 으로 시작하면 주석 처리되어 커밋에 보이지 않습니다.
  • 다음 사이트를 참고하여 본인에 맞게 룰을 작성하면 됩니다.

🔗 Commit message rule

  • Conventional Commits 1.0.0
  • 좋은 git 커밋 메시지를 작성하기 위한 7가지 약속
  • Writing Meaningful Git Commit Messages
  ################
  # <타입>: <제목> 의 형식으로 제목을 아래 공백줄에 작성
  # 제목은 50자 이내, 변경사항 명확히 작성, 끝에 마침표 금지

  # 바로 아래 공백은 지우지 마세요 (제목과 본문의 분리를 위함)

  ################
  # 본문(구체적인 내용)을 아랫줄에 작성

  ################
  # 꼬리말(footer)을 아랫줄에 작성

  ################
  # FEAT : 새로운 기능 추가
  # FIX : 버그 수정
  # DOCS : 문서 수정
  # TEST : 테스트 코드 추가
  # REFACTOR : 코드 리팩토링
  # STYLE : 코드 의미에 영향을 주지 않는 변경사항
  # CHORE : 빌드 부분 혹은 패키지 매니저 수정사항
  # CONTENT : 글 업데이트
  # DEPLOY : 배포
  # BUILD : 빌드
  ################

commit message template 설정하기

아래 명령어를 terminal 에 입력합니다.

git config --global commit.template .gitmessage.txt

template을 적용하면 git commit 할때마다 지정한 파일이 vi 로 나옵니다.

i 를 눌러 제목 본문 꼬리말을 작성하고 esc를 누른 후에 :wq!를 입력한 후 enter를 눌러 작성한 것을 저장하여 커밋합니다.

IntelliJ에서 사용해보기

Custom commit template 사용하기

위에서 설정한 commit message template을 IntelliJ에서 commit 할때 사용할 수 있습니다.

  1. Settings > Plugins > MarketPlace 에서 해당 plugin 을 찾아 설치
  1. 설정한 commit message template 적용
  • Settings > Tools > Commit Message Template 에서 template을 적용해줍니다.
  • 위에 text를 직접 입력해도 되고, Load Template File에 만들어둔 text 파일 directory 주소를 입력해도 됩니다.

맥의 경우 파일을 찾을 때, .gitmessage.txt 파일의 경우 숨겨질 수 있기에 command + shift + . 을 누르면 숨긴 파일도 볼 수 있습니다.
