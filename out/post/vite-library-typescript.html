vite library type 정의하기: vite-plugin-dts

2024/2/2

현재 vite 환경에서 Storybook을 이용한 간단한 ui package를 만들고 있습니다.

TypeScript를 사용하여 라이브러리 프로젝트를 개발하려면 타입 정의 파일(d.ts)를 정의해줘야 하기에 vite plugin인 vite-plugin-dts 을 설치하여 정의해 주도록 합시다.

설치

// npm을 사용하는 경우
npm i -D vite-plugin-dts

// yarn을 사용하는 경우
yarn add -D vite-plugin-dts

// pnpm을 사용하는 경우
pnpm i vite-plugin-dts -D

라이브러리 설정하기

vite.config.ts 수정하기

dts를 vite.config,ts plugins 에 추가해줍니다.

insertTypesEntry 옵션을 지정하여 통해 package.json types 속성에 지정한 위치에 타입 정의 파일을 생성해주도록 합니다.

rollupOptions 에서 다음에 해당하는 내용를 작성해주도록 합니다.

  • external: 번들에 포합하지 않을 라이브러리를 dependency 명시하도록 합니다.
  • output
    • globals: 라이브러리 외부에 존재하는 dependency를 위해 번들링 시 사용될 전역 변수 명시합니다.
  import { defineConfig } from "vite"
  import react from "@vitejs/plugin-react"
  import dts from "vite-plugin-dts"
  import tsconfigPaths from "vite-tsconfig-paths"
  import path from "path"

  // https://vitejs.dev/config/
  export default defineConfig({
    plugins: [
      ...,
      dts({
        insertTypesEntry: true
      }),
      ...
    ],
    build: {
      lib: {
        entry: path.resolve(__dirname, "src/index.ts"),
        name: "coyo-ui",
        formats: ["es", "cjs"],
        fileName: format => `index.${format}.js`
      },
      rollupOptions: {
        external: ["react", "react-dom"],
        output: {
          globals: {
            react: "React",
            "react-dom": "ReactDOM"
          },
          ...
        }
      },
      ...
    }
  })

package.json 수정하기

다음과 같이 types: “./{outDir}/{typeFileName}.d.ts” 을 package.json 과 exports안에도 추가해주도록 합니다.

  • types : TypeScript 프로젝트의 타입 경로를 지정합니다.
  {
    ...
    "types": "./dist/index.d.ts",
    "main": "./dist/index.cjs.js",
    "module": "./dist/index.es.js",
    "exports": {
      ".": {
        "types": "./dist/index.d.ts",
        "import": "./dist/index.es.js",
        "require": "./dist/index.cjs.js"
      }
    },
    ...
  }

적용 결과

build를 진행하면 지정해주었던 것과 같이 dist/index.d.ts 파일이 생성된 것을 확인할 수 있습니다.
