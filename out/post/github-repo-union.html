Github Repository 하나로 합치기

2024/2/6

프로젝트를 이것저것 진행하다보니 github repository 가 많아져서 기존 commit을 남기고 관련있는 repository를 병합하려고 합니다.

새로운 Repository 생성

기존 repository들을 보관할 새로운 repository를 생성해주도록 합니다.

생성한 Repository Local과 Remote 연결하기

repo 주소를 복사해서 다음과 같이 입력하여 remote와 local을 연결해줍니다.

git init
git clone 복사한 주소

Working tree has modifications. Cannot add.

해당 오류를 방지하기 위해 이후에 readme.md 나 파일 등을 수정하여 commit 을 진행해줍니다.

기존의 Repository를 새로 생성한 Repository에 합치기

합치고자 하는 기존의 repository의 주소를 복사해줍니다. 이때 메인 브랜치명도 반드시 확인해주도록 합시다.

git subtree add --prefix=기존레포지토리명 기존레포지토리주소 기존메인브랜치명

해당 명령어를 입력하면 해당 Repository가 추가된 것을 확인할 수 있습니다.

다음 명령어를 입력하여 push를 진행하여 remote에도 적용시켜주도록 합니다.

git push origin HEAD:main

github의 remote repository를 확인해보면 다음과 같이 잘 적용된 것을 확인할 수 있습니다.
