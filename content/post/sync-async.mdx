# 동기(Synchronous)와 비동기(Asynchronous)
: 작업(프로세스)를 처리하는 방식, 작업 완료 여부에 초점

# 블록킹(Blocking)과 논블록킹(Non-Blocking)
: 프로세스 유휴 상태, 다른 요청의 작업을 처리하기 위해 현재 작업 여부(차단/대기)에 초점

| 블록킹(Blocking) | 논블록킹(Non-Blocking) |
| --- | --- |
| <img width="500" alt="image" src="https://github.com/user-attachments/assets/534407bf-a9f4-4f0f-a885-9156a9418d78"/> | <img width="500" alt="image" src="https://github.com/user-attachments/assets/024083e1-c2fc-4e23-8414-8b1223e65071"/> |
| 다른 작업이 다 수행될 때까지 현재 작업 차단 | 다른 작업과 현재 작업이 동시에 진행(현재 작업을 막지 않음 = non-blocking) |

# 동기(Synchronous)/비동기(Asynchronous) + 블록킹(Blocking)/논블록킹(Non-Blocking)
<br/>

| | 블록킹(Blocking)<br/><span className={"font-normal"}>: 호출된 대상이 제어권을 가짐</span> | 논블록킹(Non-Blocking)<br/><span className={"font-normal"}>: 호출한 대상이 제어권을 가짐</span> |

|**동기(Synchronous)**<br/>: 호출한 함수가 작업 완료 여부를 확인| <img width="360" alt="image" src="https://github.com/user-attachments/assets/60be6096-a2ee-4a6b-a080-b1cce0ac63fd"/><br/>* 코드 순차 진행<br/>* 작업이 간단하거나 양이 적은 경우에 사용된다.<br/>* 작업량이 많거나 복잡한 경우에는 시간이 오래 걸리기 때문에 비효율적이다. | <img width="360" alt="image" src="https://github.com/user-attachments/assets/aa1f156b-faca-429c-926e-a98f0fc752c5"/><br/>* 자신의 작업을 진행하고 있지만 다른 작업과의 동기를 위해서 계속해서 다른 작업의 완료 여부를 조회한다. |
|**비동기(Asynchronous)**<br/>: Callback 함수가 작업 완료 여부를 확인| <img width="360" alt="image" src="https://github.com/user-attachments/assets/aa808961-5b90-4c22-8b5b-2594b9a00afa"/><br/>* 블로킹으로 인해 결국 다른 작업이 끝날 때까지 기다려야 하기에 동기 + 블로킹 작업과 동일한 결과가 나온다. | <img width="360" alt="image" src="https://github.com/user-attachments/assets/c567926e-d881-4b97-9855-20020c89ece0"/><br/>* 자원이 충분하다면 가장 효율이 좋은 조합이다.<br/>* 순차가 보장되지 않기에 동기가 필요하지 않고 작업이 대규모이고 시간이 오래 걸리는 경우에 사용하면 좋다. |
